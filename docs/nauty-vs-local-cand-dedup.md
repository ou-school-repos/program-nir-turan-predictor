Here's the breakdown of what's running at each depth:

### Nauty levels (points 3–8 for R=9)

We build a **4-colored auxiliary graph** encoding the permutation set, then call `densenauty()` which runs **McKay's canonical labeling algorithm**:

1. **Partition refinement** (a.k.a. color refinement / 1-WL): iteratively splits vertex color classes based on neighbor counts until stable
2. **Individualization**: when refinement alone can't produce a unique labeling, it picks a vertex from the coarsest cell, fixes it, and branches
3. **Automorphism pruning**: as it explores the individualization tree, it detects automorphisms of the graph and uses them to skip symmetric branches
4. The output is a **canonical form** — isomorphic graphs always produce the identical canonical graph

We then hash that canonical graph to 128 bits and check our `FlatHashSet128`. If already seen → the current vertex set is isomorphic to one we've already fully explored → prune the entire subtree.

The 4-color auxiliary graph has `R + N + R·N + point` nodes (positions, symbols, grid cells, permutations) — the coloring ensures nauty only permutes vertices within the same type, giving us the full S_n × S_R symmetry group.

### Leaf level (point = R = 9)

**No dedup at all.** Leaves never branch, so deduplicating them would waste memory for zero gain. The leaf just:

1. Uses the **accumulated incremental `calc_step`** values (`acc_nk1`, `acc_cons`) — already computed O(R) at each parent level
2. Stores the result if it's the best `cons` seen for that `nk1` class

### Local candidate dedup (every non-leaf level)

A **2048-entry open-addressing hash table** on the stack. Before spawning a child, we check if this exact `temp` vertex was already generated by a different `(i, j, k)` combination at the same parent. This is a simple **linear-probing hash set** — not a graph isomorphism check, just exact uint64 dedup.

### Summary

| Depth        | Algorithm                        | Cost per node            | What it catches                       |
| ------------ | -------------------------------- | ------------------------ | ------------------------------------- |
| 3–8          | McKay canonical labeling (nauty) | O(n² log n) on aux graph | Full isomorphism (S_n × S_R symmetry) |
| 9 (leaf)     | None — just accumulate           | O(1)                     | N/A (no branching)                    |
| All non-leaf | Local hash dedup                 | O(1) amortized           | Duplicate children from same parent   |
